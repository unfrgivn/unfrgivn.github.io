Shared Development Toolkit

TechnologyGitHub ActionsJavaScriptPHPPython
SubheadUnified set of development tools, plugins, libraries, and CI/CD pipelines used across all applications.

Problem

  • Lack of process around tool selection lead to multiple packages for code quality, testing, and functionality.
  • Repeated use of contributed packages lead to scattered and inconsistent versions and configuration.

Goals

  • Drive all packages and shared development tools from a central repository that could be updated, documented, and governed by DevSecOps.

Criteria

  • Singular instruction set for configuration and definition of contributed packages
  • Deploys artifacts that are compatible out-of-the-box with common package managers
  • The repository itself is managed using the development tools it hosts

Outcome

  • Monorepo created with self-referencing configuration for code quality and transposition.
  • Individual components for linting, compiling, testing, common libraries, conventional commits, and more all built, tested, and deployed to read-only mirrors and package indecies.
  • Smart-detection of changes with dependency chaining used to ensure that only necessary packages were rebuilt and properly versioned based on conventional commits.
